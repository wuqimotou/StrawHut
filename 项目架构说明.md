## 1 架构设计总览

### 1.1 架构原则

StrawHut 采用 **三层架构** 设计，遵循以下核心原则：

| 原则 | 说明 |
|------|------|
| **纯本地运行** | 所有功能在用户设备上执行，无任何网络请求模块 |
| **零持久化存储** | 软件不保存任何卡片、密钥或草稿记录，关闭后内存完全清空 |
| **关注点分离** | UI 层、业务逻辑层、数据访问层严格分离 |
| **依赖倒置** | 上层依赖抽象接口，不依赖具体实现 |
| **安全优先** | 敏感数据在内存中使用后立即清理 |
| **可扩展性** | 预留 P2P 扩展接口，支持未来平滑升级 |

### 1.3 模块交互关系

```
  用户操作
     |
     v

     |
     v
[核心服务层] ────> 调用对应的业务服务（加密/解密/文件操作/草稿管理）
     |
     v

     |

     |

     |
     v
[应用层 UI] ──────> 渲染更新后的界面
```

## 2 三层架构设计

### 2.1 应用层 (Application Layer)

**职责：**
- 用户界面展示与交互
- 用户输入捕获与验证
- 调用核心服务层执行业务逻辑
- 根据服务层返回结果更新 UI 状态

**组件：**

| 组件 | 职责 | 对应页面 |
|------|------|---------|
| HomeScreen | 应用入口，提供编辑/打开/拖拽操作入口 | 首页 |
| EditorScreen | 富文本编辑器界面，工具栏，发布按钮 | 知识编辑页 |
| ReaderScreen | 解密后富文本渲染展示，元数据预览 | 知识阅读页 |
| PublishDialog | 发布/加密对话框：填写元信息、匿名切换、密钥展示、导出选项 | 发布弹窗 |
| DecryptDialog | 解密对话框：密钥输入、.key 文件上传、解密进度 | 解密弹窗 |

**技术要点：**
- 使用 Riverpod 进行状态管理，UI 组件通过 `ConsumerWidget` 或 `ConsumerStatefulWidget` 订阅状态
- UI 组件保持纯粹，不包含业务逻辑
- 所有异步操作通过 `Future` 或 Riverpod Provider 处理
- 应用配置（MaterialApp、路由、主题、国际化）在 `lib/app/` 目录下统一管理

### 2.2 核心服务层 (Core Service Layer)

**职责：**
- 封装所有业务逻辑
- 提供清晰的服务接口供应用层调用
- 处理数据转换和格式验证
- 管理内存中的运行时状态

**服务定义：**

```dart
/// 加密服务接口
abstract class ICryptoService {
  /// 生成加密密钥（CSPRNG）
  Future<GeneratedKey> generateKey();

  /// 加密知识内容（AES-256-GCM）
  Future<EncryptedContent> encryptContent({
    required String deltaJson,
    required Uint8List key,
  });

  /// 解密知识内容（AES-256-GCM）
  Future<String> decryptContent({
    required String encryptedDataBase64,
    required String ivBase64,
    required Uint8List key,
  });

  /// 清理敏感数据（内存安全）
  void clearSensitiveData();
}

/// 文件 I/O 服务接口
abstract class IFileIOService {
  /// 读取 .straw 文件
  Future<StrawFile> readStrawFile(String filePath);

  /// 写入 .straw 文件
  Future<void> writeStrawFile({
    required String content,
    required String targetPath,
  });

  /// 读取 .key 文件
  Future<KeyFile> readKeyFile(String filePath);

  /// 写入 .key 文件
  Future<void> writeKeyFile({
    required String content,
    required String targetPath,
  });

  /// 验证文件扩展名
  bool isValidStrawFile(String filePath);
  bool isValidKeyFile(String filePath);
}

/// 内存草稿管理服务接口
abstract class IDraftManager {
  /// 保存内容到内存草稿
  void saveToDraft(String deltaJson);

  /// 从内存草稿读取内容
  String? loadFromDraft();

  /// 清除内存草稿
  void clearDraft();

  /// 检查是否存在草稿
  bool hasDraft();
}

/// 完整性校验服务接口
abstract class IIntegrityService {
  /// 计算文件内容哈希
  String computeHash(String content);

  /// 验证文件完整性
  bool verifyIntegrity({
    required String content,
    required String expectedHash,
  });
}

/// 格式验证服务接口
abstract class IFormatValidator {
  /// 验证 .straw 文件格式
  ValidationResult validateStrawFormat(Map<String, dynamic> json);

  /// 验证 .key 文件格式
  ValidationResult validateKeyFormat(Map<String, dynamic> json);
}
```

### 2.3 数据层 (Data Layer)

**职责：**
- 定义数据模型并提供 JSON 序列化/反序列化能力
- 封装文件仓库操作，协调核心服务层完成文件读写
- 处理跨平台文件路径差异

**设计要点：**

| 要点 | 说明 |
|------|------|
| 无数据库 | 不使用任何本地数据库，仅使用文件系统 |
| 无缓存 | 不缓存任何文件内容，每次操作直接读取文件系统 |
| 模型与序列化 | StrawFile、KeyFile、CardMeta 等数据模型提供 `fromJson`/`toJson` 方法 |
| 文件仓库 | FileRepository 封装 FileIOService 调用，提供更高层的业务接口 |
| 异常处理 | 文件不存在、权限不足、格式错误等异常统一处理 |

**数据流：**

```
用户输入 → QuillEditor.controller 内容变化
                      ↓
           _onContentChanged() 触发
                      ↓

                      ↓
           _saveToProvider()
                      ↓
           Delta → JSON 序列化
                      ↓

                      ↓

                      ↓

                      ↓

                      ↓
```

**状态管理：**

项目使用 `riverpod_annotation` 和 `build_runner` 自动生成 Provider 代码（`.g.dart` 文件），而非手写 Provider。

```dart
// editor_provider.dart - 使用 riverpod_annotation 注解
@Riverpod(keepAlive: true)
class EditorContent extends _$EditorContent {
  /// 初始状态：空字符串表示编辑器无内容
  @override
  String build() => '';

  /// 更新编辑器内容，同时保存到草稿
  void updateContent(String deltaJson) {
    state = deltaJson;
    ref.read(draftManagerProvider).saveToDraft(deltaJson);
  }

  /// 从草稿加载内容
  void loadFromDraft() {
    final draft = ref.read(draftManagerProvider).loadFromDraft();
    if (draft != null) {
      state = draft;
    }
  }

  /// 清空编辑器内容和草稿
  void clear() {
    state = '';
    ref.read(draftManagerProvider).clearDraft();
  }
}
```

**关键设计决策：**

| 决策 | 说明 |
|------|------|
| `keepAlive: true` | 确保页面切换时状态不丢失，页面销毁时需手动清理 |
| 防抖保存（300ms） | 避免每次输入都触发 Provider 更新，防止焦点丢失和性能问题 |
| 分离 Provider 与 Controller | QuillController 在 Widget 层管理，Provider 仅管理 JSON 字符串状态 |
| 延迟加载草稿 | 使用 addPostFrameCallback 在 build 完成后加载草稿，避免工具栏 rebuild 报错 |
| flushDraft 机制 | 发布前立即刷新草稿，取消防抖等待，确保最新内容被保存 |
| 预览模式独立 Controller | PreviewPanel 使用独立的只读 QuillController，与编辑器 Controller 隔离 |

**加密服务实现：**

```dart
class CryptoService implements ICryptoService {
  // 完整性校验服务依赖（公开字段）
  final IntegrityService integrityService;

  CryptoService(this.integrityService);

  @override
  Future<GeneratedKey> generateKey() async {
    // 使用 CSPRNG 生成 32 字节随机密钥
    final random = Random.secure();
    final keyBytes = Uint8List(KEY_LENGTH_BYTES);
    for (var i = 0; i < KEY_LENGTH_BYTES; i++) {
      keyBytes[i] = random.nextInt(256);
    }
    final keyBase64 = base64Encode(keyBytes);
    return GeneratedKey(bytes: keyBytes, base64: keyBase64);
  }

  @override
  Future<EncryptedContent> encryptContent({
    required String deltaJson,
    required Uint8List key,
  }) async {
    // AES-256-GCM 加密
    final encrypter = Encrypter(AES(Key(key), mode: AESMode.gcm));
    final iv = IV.fromSecureRandom(IV_LENGTH_BYTES);  // 16 字节 IV

    // 将 Delta JSON 字符串转为 UTF-8 字节后加密
    final encrypted = encrypter.encrypt(deltaJson, iv: iv);

    return EncryptedContent(
      encryptedDataBase64: base64Encode(encrypted.bytes),
      ivBase64: base64Encode(iv.bytes),
      algorithm: ENCRYPTION_ALGORITHM_AES_256_GCM,
    );
  }

  @override
  Future<String> decryptContent({
    required String encryptedDataBase64,
    required String ivBase64,
    required Uint8List key,
  }) async {
    try {
      // 验证密钥长度是否为 32 字节
      if (key.length != KEY_LENGTH_BYTES) {
        throw CryptoException(
          '密钥长度不正确：期望 $KEY_LENGTH_BYTES 字节，实际 ${key.length} 字节',
          code: 'INVALID_KEY_LENGTH',
        );
      }

      // Base64 解码 IV 和密文数据
      final ivBytes = base64Decode(ivBase64);
      final encryptedBytes = base64Decode(encryptedDataBase64);

      // 创建 AES-256-GCM 解密器
      final encrypter = Encrypter(AES(Key(key), mode: AESMode.gcm));

      // 执行解密操作，GCM 模式自动验证 MAC
      final decrypted = encrypter.decrypt(
        Encrypted(encryptedBytes),
        iv: IV(ivBytes),
      );

      return decrypted;
    } on CryptoException {
      // 已知的加密异常，直接向上抛出
      rethrow;
    } catch (e) {
      // 其他异常统一包装为 CryptoException
      throw CryptoException(
        '解密失败：可能是密钥错误或文件已损坏。详情：$e',
        code: 'DECRYPTION_FAILED',
      );
    }
  }

  @override
  void clearSensitiveData() {
    // 当前实现为无状态设计，没有内部持有敏感数据引用。
    // 调用方应自行清理持有的密钥引用，例如：
    //   MemoryUtils.wipeBytes(keyBytes);
    //   keyBytes = null;
  }
}
```

```dart

enum ReaderStatus {
  loading,    // 正在加载文件
  error,      // 文件加载失败
  metaOnly,   // 文件已加载但尚未解密
  decrypted,  // 解密成功，展示内容
}

class _ReaderScreenState extends ConsumerState<ReaderScreen> {
  ReaderStatus _status = ReaderStatus.loading;
  String? _decryptedContent;
  StrawFile? _strawFile;
  String? _errorMessage;
  bool _hasShownDecryptDialog = false;

  /// 加载知识卡片文件
  Future<void> _loadFile() async {
    // 从路由参数中获取文件路径
    final filePath = GoRouterState.of(context).uri.queryParameters['path'];
    
    // 使用 CurrentCard Provider 加载文件
    await ref.read(currentCardProvider.notifier).loadFile(filePath);
    
    final cardState = ref.read(currentCardProvider);
    final strawFile = cardState.valueOrNull;
    
    if (strawFile != null) {
      _strawFile = strawFile;
      _status = ReaderStatus.metaOnly;
      _showDecryptDialog(); // 自动弹出解密对话框
    }
  }

  /// 弹出解密对话框
  void _showDecryptDialog() {
    if (_hasShownDecryptDialog || _strawFile == null) return;
    _hasShownDecryptDialog = true;

    DecryptDialog.show(
      context,
      strawFile: _strawFile!,
      onDecryptSuccess: (deltaJson) {
        if (mounted) {
          setState(() {
            _decryptedContent = deltaJson;
            _status = ReaderStatus.decrypted;
          });
        }
      },
    );
  }
}
```

**阅读器状态转换：**

```
ReaderStatus.loading
    │
    ├── [文件加载成功] → ReaderStatus.metaOnly
    │                        │
    │                        ├── [用户取消解密] → 保持 metaOnly
    │                        │
    │                        └── [解密成功] → ReaderStatus.decrypted
    │                                             │
    │                                             └── [点击"重新解密"] → metaOnly
    │
    └── [文件加载失败] → ReaderStatus.error
                              │
                              └── [点击"重试"] → loading
```

**关键设计决策：**

| 决策 | 说明 |
|------|------|
| 不使用 @riverpod ReaderState | ReaderScreen 自身管理状态即可，无需独立 Provider |
| CurrentCard Provider | 用于文件加载操作，返回 AsyncValue\<StrawFile\> |
| ReaderStatus 四状态枚举 | 明确区分加载中/错误/仅元数据/已解密四种状态 |
| 完整性校验重建 StrawFile | 因发布时 integrity.hash 为空占位，校验时也需重建空 hash 的 StrawFile |
| MemoryUtils.wipeBytes | 解密后逐字节置零密钥字节数组，比单纯置 null 更安全 |
| 防止重复弹出对话框 | _hasShownDecryptDialog 标记确保对话框只弹出一次 |
| 回调模式而非 Provider 模式 | DecryptDialog 使用 onDecryptSuccess 回调，与 ReaderScreen 解耦 |

### 4.4 日期时间处理机制

**设计原则：**

| 原则 | 说明 |

**实现细节：**

**存储端（publish_dialog.dart）：**
```dart
// 使用 UTC 时间存储
final now = DateTime.now().toUtc();
final meta = CardMeta(

```

**显示端（_formatDate 方法）：**
```dart
// 解析 UTC 时间并转换为本地时区后提取日期
String _formatDate(String isoDate) {

    final dateTime = DateTime.parse(isoDate).toLocal();
    return '${dateTime.year}-${dateTime.month.toString().padLeft(2, '0')}-${dateTime.day.toString().padLeft(2, '0')}';

    return isoDate;
  }
}
```

### 4.5 内存草稿模块

**模块位置：** `lib/core/draft/`

**设计要点：**

| 要点 | 说明 |
|------|------|
| 纯内存存储 | 草稿仅保存在运行内存中，不写入磁盘 |
| 关闭即消失 | 软件关闭后草稿自动清空（无需手动清理） |
| 单次会话有效 | 草稿只在当前应用生命周期内有效 |
| 可主动清除 | 用户发布后可选择清除草稿 |

**实现：**

```dart
class DraftManager implements IDraftManager {
  String? _draftContent;  // 仅保存在内存中

  @override
  void saveToDraft(String deltaJson) {
    _draftContent = deltaJson;
  }

  @override
  String? loadFromDraft() {
    return _draftContent;
  }

  @override
  void clearDraft() {
    _draftContent = null;  // 释放引用，等待 GC
  }

  @override
  bool hasDraft() {
    return _draftContent != null && _draftContent!.isNotEmpty;
  }
}
```

## 5 文件格式规范

### 5.1 .straw 知识卡片文件格式

#### 5.1.1 文件概述

`.straw` 是 StrawHut 定义的结构化知识卡片文件格式，采用 JSON 格式封装。每个 `.straw` 文件都是一张独立的知识卡片，包含加密后的知识内容和公开可见的元数据。

#### 5.1.2 完整文件结构

```json
{
  "format_version": "1.0.0",
  "meta": {
    "publisher_alias": "Anonymous",
    "publish_date": "2026-05-01T12:00:00Z",
    "title": "网络安全入门指南",
    "tags": ["安全", "入门", "笔记"],
    "description": "一份关于网络安全基础知识的入门笔记",
    "is_anonymous": true,
    "custom_annotations": {}
  },
  "content": {
    "encrypted_data": "Base64EncodedEncryptedContent...",
    "encryption_algorithm": "AES-256-GCM",
    "iv": "Base64EncodedIV..."
  },
  "integrity": {
    "hash": "sha256:abcdef1234567890...",
    "hash_algorithm": "SHA-256"
  }
}
```

#### 5.1.3 字段详细说明

**顶层结构：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `format_version` | string | 是 | 文件格式版本号，语义化版本 `major.minor.patch` |
| `meta` | object | 是 | 公开可见的元数据（未加密） |
| `content` | object | 是 | 加密后的知识内容 |
| `integrity` | object | 是 | 文件完整性校验信息 |

**meta 对象：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `publisher_alias` | string | 是 | 发布者代号，匿名模式下固定为 "Anonymous" |
| `publish_date` | string | 是 | 发布日期时间，ISO 8601 格式（UTC），存储时使用 UTC 时间，显示时自动转换为本地时区 |
| `title` | string | 是 | 知识卡片标题 |
| `tags` | string[] | 否 | 标签列表，最多 10 个标签，每个标签最长 20 字符，默认为空数组 `[]` |
| `description` | string | 否 | 简要描述，最长 200 字符，未解密时可见，为 null 时不输出到 JSON |
| `is_anonymous` | boolean | 是 | 是否为纯匿名模式 |
| `custom_annotations` | object | 否 | 自定义键值对标注，键最长 50 字符，值最长 500 字符，为 null 时不输出到 JSON |

**content 对象：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `encrypted_data` | string | 是 | 加密后的知识内容，Base64 编码（URL-safe） |
| `encryption_algorithm` | string | 是 | 加密算法标识，当前固定为 `AES-256-GCM` |
| `iv` | string | 是 | 初始化向量，Base64 编码（16 字节） |

**integrity 对象：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `hash` | string | 是 | 文件完整性校验哈希，格式为 `算法:哈希值` |
| `hash_algorithm` | string | 是 | 哈希算法标识，当前固定为 `SHA-256` |

#### 5.1.4 知识内容格式（加密前）

解密后的知识内容为 **Delta JSON 格式**（Quill 编辑器原生格式）：

```json
{
  "ops": [
    { "insert": "网络安全入门指南\n", "attributes": { "header": 1 } },
    { "insert": "这是一份关于" },
    { "insert": "网络安全", "attributes": { "bold": true } },
    { "insert": "的基础知识笔记。\n" },
    { "insert": "\n", "attributes": { "header": 2 } },
    { "insert": "基本概念\n", "attributes": { "header": 3 } },
    { "insert": "• 防火墙\n• 入侵检测\n• 加密通信\n", "attributes": { "list": "bullet" } },
    { "insert": "\n", "attributes": { "header": 3 } },
    { "insert": "代码示例：\n", "attributes": { "header": 3 } },
    { "insert": "const key = crypto.randomBytes(32);\n", "attributes": { "code-block": "javascript" } }
  ]
}
```

**Delta 格式优势：**
- 与 flutter_quill 编辑器原生兼容，无需转换
- JSON 格式可读性好，便于调试
- 支持未来扩展新操作类型
- 序列化后加密，安全性有保障

#### 5.1.5 版本兼容策略

| 版本策略 | 说明 |
|---------|------|
| **主版本号 (Major)** | 不兼容的格式变更，旧版本无法读取新版本文件 |
| **次版本号 (Minor)** | 向后兼容的功能新增，旧版本可忽略新字段读取 |
| **修订号 (Patch)** | 向后兼容的 Bug 修复 |

**兼容性处理逻辑：**

```
读取文件时：
  1. 解析 format_version
  2. 比较主版本号：
     - 相同 → 正常读取
     - 当前主版本 < 文件主版本 → 提示"需要更新软件"，拒绝读取
     - 当前主版本 > 文件主版本 → 兼容模式读取（向前兼容）
  3. 忽略未知字段（向后兼容）
```

#### 5.1.6 文件大小建议

| 内容类型 | 建议限制 | 说明 |
|---------|---------|------|
| 纯文本卡片 | 无硬性限制，建议 < 1MB | 1MB 纯文本约 50 万字 |
| 含图片卡片 | 建议 < 10MB | 图片采用 Base64 嵌入时体积膨胀约 33% |
| 超大知识库 | 建议拆分为多张卡片 | 单张卡片过大影响加密/解密性能 |

### 5.2 .key 密钥文件格式

#### 5.2.1 文件概述

`.key` 是 StrawHut 定义的密钥文件格式，用于将加密密钥独立保存为文件。`.key` 文件可与 `.straw` 文件分开传输和存储，用户通过上传 `.key` 文件实现快速解密。

**设计原则：**
- `.key` 文件独立于 `.straw` 文件，不随 `.straw` 文件一起生成
- 用户自主选择是否导出，避免密钥与加密内容一起泄露
- 采用 JSON 格式，便于解析和扩展
- 包含关联元信息，方便用户识别密钥用途

#### 5.2.2 完整文件结构

```json
{
  "format_version": "1.0.0",
  "key_metadata": {
    "key_id": "k_2026050112000000_a3f7b2c1",
    "created_at": "2026-05-01T12:00:00Z",
    "associated_card_title": "网络安全入门指南",
    "key_algorithm": "AES-256-GCM",
    "key_length_bits": 256
  },
  "key_data": {
    "key_base64": "xK9mP2vR7wN4jQ8tL5yH3bF6dA1cE0gU6sZ9oI2eM=",
    "encoding": "base64"
  },
  "integrity": {
    "hash": "sha256:fedcba0987654321...",
    "hash_algorithm": "SHA-256"
  }
}
```

> **注意**：可选字段（`associated_card_id`、`notes`）为 null 时不会输出到 JSON。

#### 5.2.3 字段详细说明

**顶层结构：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `format_version` | string | 是 | 文件格式版本号，与 .straw 版本独立管理 |
| `key_metadata` | object | 是 | 密钥元信息 |
| `key_data` | object | 是 | 密钥数据 |
| `integrity` | object | 是 | 文件完整性校验 |

**key_metadata 对象：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `key_id` | string | 是 | 密钥唯一标识，格式：`k_时间戳_随机标识` |
| `created_at` | string | 是 | 密钥创建时间，ISO 8601 格式（UTC） |
| `associated_card_title` | string | 否 | 关联的卡片标题（方便用户识别） |
| `associated_card_id` | string | 否 | 关联的卡片标识（当前版本为 null，预留） |
| `key_algorithm` | string | 是 | 密钥对应的加密算法标识 |
| `key_length_bits` | number | 是 | 密钥长度（比特） |
| `notes` | string | 否 | 用户自定义备注，最长 200 字符 |

**key_data 对象：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `key_base64` | string | 是 | 密钥的 Base64 编码字符串 |
| `encoding` | string | 是 | 编码方式，固定为 `base64` |

**integrity 对象：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `hash` | string | 是 | 文件完整性校验哈希 |
| `hash_algorithm` | string | 是 | 哈希算法标识 |

#### 5.2.4 密钥 ID 生成规则

```
格式：k_{YYYYMMDDHHmmssSSS}_{8位随机十六进制}

示例：k_20260501120000000_a3f7b2c1

说明：
  - 时间戳精确到毫秒（17 位）
  - 随机标识 8 位十六进制（32 位熵）
  - 总长度固定，便于解析
```

#### 5.2.5 安全考虑

| 安全措施 | 说明 |
|---------|------|
| 完整性校验 | `.key` 文件包含 SHA-256 哈希，防止密钥被篡改 |
| 独立存储 | 密钥文件与加密内容文件分离存储和传输 |
| 明文存储 | 密钥文件本身不加密（用户自行保管安全性） |
| 扩展名固定 | 固定使用 `.key` 扩展名，便于识别 |
| 用户提示 | 导出时明确提示用户妥善保管 `.key` 文件 |

#### 5.2.6 为什么不加密 .key 文件

| 方案 | 优点 | 缺点 | StrawHut 选择 |
|------|------|------|--------------|
| 明文 JSON | 简单、可直接使用、用户可读 | 泄露即失效 | **推荐** |
| 密码加密 | 额外安全层 | 需要用户记住密码、增加复杂度 | 不采用 |
| 自定义二进制 | 体积更小 | 可读性差、开发成本高 | 不采用 |

**理由：**
- StrawHut 的核心理念是"密钥即权限"，密钥本身需要用户妥善保管
- 增加密码保护会引入额外的密码管理复杂度，与产品定位冲突
- 用户可通过操作系统层面的文件加密（如 NTFS EFS、macOS FileVault）保护 `.key` 文件

### 6.2 解密读取数据流

```
+-----------------------------------------------------------------------------+
|                          解密读取 - 数据流                                   |
+-----------------------------------------------------------------------------+
|                                                                             |
|  [文件系统] → .straw 文件                                                    |
|       │                                                                     |
|       ▼                                                                     |
|  [FileIOService.readStrawFile()]                                            |
|       │                                                                     |
|       │ 文件内容 (JSON 字符串)                                               |
|       ▼                                                                     |
|  [JSON.parse()]                                                             |
|       │                                                                     |
|       │ Map<String, dynamic>                                                |
|       ▼                                                                     |
|  [FormatValidator.validateStrawFormat()]                                     |
|       │                                                                     |
|       │ 验证通过 → StrawFile 对象                                            |
|       ▼                                                                     |
|  [展示 Meta 信息] → ReaderScreen 展示元数据预览                               |

|       │                                                                     |
|       ▼                                                                     |
|  [QuillEditor.readonly] → 渲染富文本内容                                      |
|                                                                             |

|  [CryptoService.clearSensitiveData()] → 清除内存中的密钥                      |

```

## 7 安全架构设计

### 7.1 安全设计原则

| 原则 | 说明 |
|------|------|
| **零信任本地** | 所有输入数据均视为不可信，需经过格式验证 |
| **最小权限** | 敏感数据仅在需要时存在于内存中，使用后立即清理 |
| **纵深防御** | 多层安全机制（加密 + 完整性校验 + 格式验证） |
| **安全默认** | 默认使用最强安全配置（AES-256-GCM、256 位密钥） |

### 7.2 内存安全设计

#### 7.2.1 敏感数据清单

| 数据类型 | 存在位置 | 生命周期 | 清理时机 |
|---------|---------|---------|---------|
| 加密密钥 (bytes) | CryptoService 局部变量 | 加密/解密操作期间 | 操作完成后立即置 null |
| 加密密钥 (base64) | PublishDialog UI 变量 | 对话框展示期间 | 用户确认后清除 |
| 明文内容 (Delta JSON) | EditorScreen 内存 | 编辑期间 | 发布后保留（用户可继续使用） |
| 解密后的明文 | ReaderScreen 内存 | 阅读期间 | 关闭页面后释放 |
| 内存草稿 | DraftManager._draftContent | 应用生命周期内 | 主动清除或应用退出 |

#### 7.2.2 内存清理策略

```dart
/// 内存清理工具
class MemoryUtils {
  MemoryUtils._();  // 私有构造函数，防止实例化

  /// 清除 Uint8List 中的敏感字节
  static void wipeBytes(Uint8List bytes) {
    for (var i = 0; i < bytes.length; i++) {
      bytes[i] = 0;  // 逐字节置零
    }
  }

  /// 安全清除 Map 中的所有值
  static void secureClearMap(Map<String, dynamic> map) {
    final keysToRemove = map.keys.toList();
    for (final key in keysToRemove) {
      final value = map[key];
      if (value is Uint8List) {
        wipeBytes(value);
      } else if (value is List<int>) {
        for (var i = 0; i < value.length; i++) {
          value[i] = 0;
        }
      }
      map[key] = null;
    }
  }
}
```

**Dart 内存安全的局限性：**

| 限制 | 说明 | 缓解措施 |
|------|------|---------|
| GC 不可控 | Dart 使用自动垃圾回收，无法强制立即回收 | 引用置 null，降低被扫描到的概率 |
| 字符串不可变 | Dart 字符串不可变，无法原地修改 | 使用 Uint8List 存储密钥而非 String |
| 内存快照 | 系统级内存快照可能捕获敏感数据 | 应用级无法完全防护，需依赖操作系统安全 |

### 7.3 匿名发布安全设计

#### 7.3.1 匿名标识

匿名模式下，`publisher_alias` 字段固定为 `"Anonymous"`，不附加任何随机后缀或标识符。

```dart
// publish_dialog.dart 中的实现
final publisherAlias = isAnonymous ? 'Anonymous' : formState.publisherAlias!;
```

#### 7.3.2 匿名模式处理逻辑

```
匿名模式开启时：
  1. publisher_alias = "Anonymous"  // 固定常量
  2. custom_annotations 中过滤可能暴露身份的信息
  3. 不收集任何设备信息、IP、时间戳（除 publish_date 外）
  4. 不记录用户行为日志

匿名模式关闭时：
  1. publisher_alias = 用户手动输入的代号
  2. 可自由填写 custom_annotations
  3. 同样不收集设备信息（纯本地运行保证）
```

### 7.4 输入安全设计

#### 7.4.1 输入验证清单

| 输入来源 | 验证项 | 处理方式 |
|---------|--------|---------|
| .straw 文件 | JSON 格式、必填字段、版本号、文件大小 | 格式错误 → 拒绝读取并提示 |
| .key 文件 | JSON 格式、必填字段、哈希校验 | 验证失败 → 拒绝使用并提示 |
| 密钥输入 | Base64 格式、长度验证（约 43 字符） | 格式错误 → 提示输入错误 |
| 元数据输入 | 标题非空、长度限制、标签数量和长度 | 超限 → 提示修改 |
| 图片 Base64 | 文件大小限制 | 超限 → 提示图片过大 |

### 7.5 安全威胁应对矩阵

| 威胁 | 攻击方式 | StrawHut 防护措施 | 剩余风险 |
|------|---------|------------------|---------|
| 密钥泄露 | 用户分享密钥给未授权人员 | 用户教育提示，.key 文件独立存储 | 低（用户行为无法控制） |
| .key 文件泄露 | 文件被他人获取 | 独立存储、完整性校验 | 低 |
| 文件篡改 | 修改 .straw 文件内容 | SHA-256 完整性校验 + GCM Auth Tag | 极低 |
| 内存泄露 | 恶意程序扫描内存 | 使用后清除敏感数据引用 | 中（Dart GC 不可控） |
| 侧信道攻击 | 通过时序分析获取密钥 | 标准库实现，无明显时序漏洞 | 低 |
| 钓鱼文件 | 恶意构造的 .straw 文件 | 格式验证、异常结构拦截 | 低 |
| 重放攻击 | 使用旧 .straw 文件 | 无状态协议，不适用 | 不适用 |

## 8 P2P 扩展接口预留

### 8.1 设计原则

| 原则 | 说明 |
|------|------|
| **接口隔离** | P2P 功能通过独立接口定义，不影响核心本地功能 |

## 9 错误处理与容错设计

### 9.1 错误分类

| 错误类型 | 场景 | 处理方式 |
|---------|------|---------|
| **用户操作错误** | 输入格式错误、文件选择取消 | 提示用户，允许重试 |
| **文件格式错误** | .straw 文件结构不完整、版本不兼容 | 明确错误信息，拒绝处理 |
| **加密错误** | 密钥错误、加密数据损坏 | 提示"密钥错误或文件已损坏" |
| **完整性校验失败** | 文件被篡改 | 提示"文件完整性校验失败，可能被篡改" |
| **文件系统错误** | 权限不足、磁盘空间不足 | 提示系统级错误信息 |
| **应用异常** | 未知运行时错误 | 通用错误提示，记录日志（仅本地） |

## 10 架构约束与最佳实践

### 10.1 架构约束

| 约束 | 规则 | 原因 |
|------|------|------|
| **无网络请求** | 禁止引入任何 HTTP/WebSocket 相关依赖 | 产品定位为纯本地工具 |
| **零持久化存储** | 禁止使用 SQLite、SharedPreferences 等持久化方案 | 关闭后不留痕迹 |
| **不存储用户数据** | 不记录编辑历史、阅读历史、密钥记录 | 隐私保护要求 |
| **单向依赖** | 应用层 → 核心服务层 → 数据层，禁止逆向依赖 | 保持架构清晰 |
