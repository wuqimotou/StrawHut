## 执行计划总览

本项目采用 **分阶段、分模块、分层级** 的开发策略。每个任务明确指定执行智能体、优先级、依赖关系和验收标准。

### 阶段规划

| 阶段 | 名称 | 范围 | 目标平台 |
|------|------|------|---------|
| **Phase 0** | 项目初始化 | 项目骨架、依赖配置、基础架构 | Windows |
| **Phase 1** | 核心服务层 | 加密服务、文件 I/O、完整性校验、草稿管理 | Windows |
| **Phase 2** | 数据层 | 数据模型定义、文件格式解析 | Windows |
| **Phase 3** | 应用层 - 编辑器 | 富文本编辑器页面、工具栏、预览 | Windows |
| **Phase 4** | 应用层 - 发布/解密 | 发布对话框、解密对话框、密钥管理 | Windows |
| **Phase 5** | 应用层 - 阅读器 | 阅读器页面、元数据预览、富文本渲染 | Windows |
| **Phase 6** | 测试与优化 | 单元测试、集成测试、性能优化 | Windows |
| **Phase 7** | 跨平台扩展 | Android/iOS 适配、平台特性集成 | Android/iOS |

### 任务 1.5: 加密服务实现

**优先级**: P0 (核心)
**依赖**: 任务 1.3, 1.4

**执行内容**:

**文件**: `lib/core/crypto/crypto_service.dart`

实现 `ICryptoService` 接口：

1. **generateKey()**: 
   - 使用 `Random.secure()` 生成 32 字节随机密钥
   - 返回 `GeneratedKey(bytes, base64)`

2. **encryptContent({deltaJson, key})**:
   - 使用 `encrypt` 包的 AES-256-GCM 模式
   - 生成 16 字节安全随机 IV
   - 返回 `EncryptedContent`

3. **decryptContent({encryptedDataBase64, ivBase64, key})**:
   - 使用相同 AES-256-GCM 模式解密
   - 解密失败抛出 `CryptoException`
   - 返回 Delta JSON 字符串

4. **clearSensitiveData()**:
   - 将所有敏感引用置 null
   - 调用 `MemoryUtils.wipeBytes()` 清理字节数组

**关键实现注意**:
- 使用 `encrypt` 包的高层 API，不直接操作 pointycastle
- IV 使用 `IV.fromSecureRandom(16)`
- 异常处理：解密失败时抛出明确的 `CryptoException`

**验收标准**:
- [ ] 加密解密可逆测试通过
- [ ] 错误密钥解密时抛出异常
- [ ] 密钥生成为 32 字节
- [ ] IV 为 16 字节
- [ ] clearSensitiveData 后敏感引用为 null
- [ ] 单元测试覆盖率 100%
